# TCP Concurrent 实现

**类别**: Mihomo 实现机制 | **模块**: TCP 并发优化详解

## 概述

TCP Concurrent（TCP 并发）是 mihomo 的连接优化功能，通过并发尝试多个服务器地址，选择最快的连接。

## 配置

```yaml
# TCP 并发配置
tcp-concurrent: true    # 启用 TCP 并发
```

## 工作原理

### Happy Eyeballs 算法

TCP Concurrent 基于 Happy Eyeballs 算法:

```
┌─────────────────────────────────────────────────────────────────────┐
│                    TCP Concurrent 工作流程                           │
└─────────────────────────────────────────────────────────────────────┘

DNS 解析返回多个地址
    │
    ▼
┌─────────────────────┐
│  地址列表           │
│  server.com         │
│  ├─ 1.2.3.4         │
│  ├─ 5.6.7.8         │
│  └─ IPv6 地址       │
└─────────────────────┘
    │
    ▼
┌─────────────────────┐
│  并发发起连接       │
│                     │
│  Socket1 → 1.2.3.4  │
│  Socket2 → 5.6.7.8  │ (延迟 250ms)
│  Socket3 → IPv6     │ (延迟 250ms)
└─────────────────────┘
    │
    ▼
┌─────────────────────┐
│  选择最快连接       │
│                     │
│  Socket1 最快       │
│  关闭其他 Socket    │
│  使用 Socket1       │
└─────────────────────┘
```

### RFC 8305 Happy Eyeballs v2

mihomo 实现遵循 RFC 8305:

```
┌─────────────────────────────────────────────────────────────────────┐
│                    Happy Eyeballs v2 (RFC 8305)                      │
└─────────────────────────────────────────────────────────────────────┘

关键参数：
┌─────────────────────────────────────────────────────────────────────┐
│  Connection Attempt Delay: 250ms                                   │
│    - 第一个连接尝试后等待 250ms                                     │
│    - 然后开始第二个尝试                                             │
│                                                                      │
│  IPv6 First:                                                        │
│    - 优先尝试 IPv6 地址                                             │
│    - IPv4 作为后备                                                  │
│                                                                      │
│  Preferred Address Family:                                          │
│    - 根据历史成功率选择首选地址族                                    │
│                                                                      │
│  Connection Attempt Timeout:                                        │
│    - 单个尝试超时后继续下一个                                        │
└─────────────────────────────────────────────────────────────────────┘
```

## 并发连接流程

```
┌─────────────────────────────────────────────────────────────────────┐
│                        并发连接流程                                  │
└─────────────────────────────────────────────────────────────────────┘

1. DNS 解析
   ┌─────────────────────────────────────────────────────────────────┐
   │  解析 server.example.com                                        │
   │  结果:                                                          │
   │  IPv6: 2001:db8::1, 2001:db8::2                                │
   │  IPv4: 1.2.3.4, 5.6.7.8                                        │
   └─────────────────────────────────────────────────────────────────┘

2. 地址排序
   ┌─────────────────────────────────────────────────────────────────┐
   │  根据历史成功率排序                                              │
   │  如果 IPv6 历史好: IPv6 地址在前                                 │
   │  如果 IPv4 历史好: IPv4 地址在前                                 │
   └─────────────────────────────────────────────────────────────────┘

3. 连接尝试
   ┌─────────────────────────────────────────────────────────────────┐
   │  T=0ms:     尝试 地址1                                          │
   │  T=250ms:   尝试 地址2                                          │
   │  T=500ms:   尝试 地址3                                          │
   │  ...                                                            │
   └─────────────────────────────────────────────────────────────────┘

4. 选择最快
   ┌─────────────────────────────────────────────────────────────────┐
   │  地址2 最先完成连接                                              │
   │  使用 地址2 的 socket                                           │
   │  关闭其他正在尝试的 socket                                       │
   └─────────────────────────────────────────────────────────────────┘

5. 连接完成
   ┌─────────────────────────────────────────────────────────────────┐
   │  使用选定的连接                                                  │
   │  更新历史成功率                                                  │
   └─────────────────────────────────────────────────────────────────┘
```

## 性能优势

### 减少连接延迟

```
┌─────────────────────────────────────┐
│       连接延迟对比                    │
├─────────────────────────────────────┤
│                                      │
│  无并发:                             │
│  地址1 失败 (300ms)                  │
│  地址2 连接 (200ms)                  │
│  总延迟: 500ms                       │
│                                      │
│  有并发:                             │
│  地址1 失败 (300ms)                  │
│  地址2 连接 (200ms)                  │
│  地址2 在 250ms 开始                 │
│  总延迟: 250+200=450ms               │
│                                      │
│  优化效果: 减少 50ms                 │
│                                      │
└─────────────────────────────────────┘
```

### IPv4/IPv6 快速切换

```
┌─────────────────────────────────────┐
│       IPv4/IPv6 切换                 │
├─────────────────────────────────────┤
│                                      │
│  场景: IPv6 连接慢或不通             │
│                                      │
│  无并发:                             │
│  IPv6 失败 (等待超时)                │
│  IPv4 连接                           │
│  总延迟: 很长                        │
│                                      │
│  有并发:                             │
│  IPv6 尝试 + IPv4 后备               │
│  IPv4 可能更快                       │
│  总延迟: 明显减少                    │
│                                      │
└─────────────────────────────────────┘
```

## 配置示例

### 基础配置

```yaml
# 启用 TCP 并发
tcp-concurrent: true
```

### 配合 IPv6

```yaml
# TCP 并发 + IPv6
tcp-concurrent: true
ipv6: true            # 启用 IPv6
```

### 配合 Unified Delay

```yaml
# TCP 并发 + Unified Delay
tcp-concurrent: true
unified-delay: true   # 统一延迟计算
```

## 实现细节

### 并发连接数

mihomo 内部控制并发连接数:
- 默认最多同时尝试 2-3 个连接
- 按时间间隔依次启动
- 任意一个成功即使用

### 资源管理

并发尝试会创建多个 socket:
- 成功后立即关闭其他
- 失败后清理资源
- 避免资源泄漏

### 历史记录

mihomo 记录地址成功率:
- 用于下次优先排序
- IPv4/IPv6 成功次数
- 自动调整优先级

## 适用场景

| 场景 | 效果 |
|------|------|
| 多 IP 负载均衡 | 好 |
| IPv4/IPv6 双栈 | 好 |
| DNS 返回多地址 | 好 |
| 单地址 | 无效果 |

## 注意事项

### 资源开销

TCP Concurrent 会:
- 创建多个临时 socket
- 消耗额外内存
- 增加短暂 CPU 使用

但开销可控，因为:
- 并发数有限
- 成功后立即清理
- 现代系统资源充足

### 与其他功能配合

配合 Unified Delay 效果更好:

```yaml
tcp-concurrent: true
unified-delay: true

# Unified Delay 确保测速准确
# TCP Concurrent 确保连接快速
```
