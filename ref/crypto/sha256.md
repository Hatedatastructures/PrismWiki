# SHA-256

**类别**: 密码学

## 概述

SHA-256（Secure Hash Algorithm 256-bit）是 SHA-2 家族中的加密哈希函数，由美国国家标准与技术研究院（NIST）在 2001 年发布（FIPS 180-2），并在 2015 年更新为 FIPS 180-4。SHA-256 生成 256 位（32 字节）的哈希值，是目前最广泛使用的加密哈希函数之一。

SHA-256 是 SHA-2 家族的成员，该家族包括 SHA-224、SHA-256、SHA-384、SHA-512、SHA-512/224 和 SHA-512/256。SHA-256 使用 32 位运算，适合 32 位处理器；SHA-512 使用 64 位运算，适合 64 位处理器。SHA-256 在大多数现代处理器上都有良好的性能平衡。

SHA-256 的设计基于 Merkle-Damgård 结构，这是一种经典的哈希函数构造方法。它将输入消息分成固定大小的块，通过压缩函数逐块处理，最终输出固定长度的哈希值。压缩函数使用消息扩展和轮函数，将输入块与当前哈希状态混合。

SHA-256 在密码学中有广泛的应用。它用于：
- 数据完整性校验：验证文件或数据是否被篡改
- 数字签名：作为 DSA、ECDSA、RSA-PSS 签名算法的哈希组件
- 密钥派生：作为 HKDF、PBKDF2 的底层哈希函数
- 消息认证：作为 HMAC-SHA256 的底层哈希函数
- 区块链：比特币和许多区块链使用 SHA-256 作为核心哈希算法
- TLS 协议：TLS 1.2 和 TLS 1.3 使用 SHA-256 作为记录层 MAC

SHA-256 的安全性经过了广泛的密码学分析。它的设计借鉴了 SHA-1 的经验，避免了 SHA-1 的已知弱点。目前，没有已知的实际攻击能够破解 SHA-256 的安全性。理论上，破解 SHA-256 需要约 2^128 次操作（碰撞攻击）或约 2^256 次操作（原像攻击），这远超任何实际计算能力。

SHA-256 的主要优势包括：
- 高安全性：256 位输出提供 128 位碰撞抵抗和 256 位原像抵抗
- 广泛支持：几乎所有加密库和协议都支持 SHA-256
- 硬件加速：现代处理器提供 SHA-NI 指令集加速
- 标准化：NIST 标准，被政府和企业广泛采纳

SHA-256 的实现相对成熟，各大加密库（如 OpenSSL、BoringSSL、CryptoAPI）都提供了经过验证的实现。现代处理器（Intel、AMD、ARM）也提供了 SHA 扩展指令，可以显著加速 SHA-256 计算。

在 TLS 1.3 中，SHA-256 是默认的哈希函数。TLS_AES_128_GCM_SHA256 和 TLS_CHACHA20_POLY1305_SHA256 都使用 SHA-256 作为握手哈希和 HKDF 的底层哈希函数。TLS 1.3 的密钥派生完全基于 HKDF-SHA256。

比特币是 SHA-256 最著名的应用之一。比特币使用 SHA-256 进行工作量证明（挖矿）、交易哈希和地址生成。比特币的双重 SHA-256（SHA256(SHA256(data)))）用于进一步增加安全性。

在代理服务器和网络协议实现中，SHA-256 用于多种场景：密钥派生、消息认证、数据完整性校验等。Prism 在 Reality 协议的密钥派生、ShadowTLS 的密钥生成等多个场景中使用 SHA-256。SHA-256 提供了安全、高效的哈希计算基础。

总的来说，SHA-256 是现代密码学中最重要的哈希函数之一。它的高安全性、广泛支持和标准化地位使其成为各种安全协议的首选哈希算法。理解 SHA-256 的工作原理对于正确实现和使用密码学系统至关重要。

## 算法原理

### Merkle-Damgård 结构

SHA-256 基于 Merkle-Damgård 结构，这是一种经典的哈希函数构造方法。

**基本原理**

Merkle-Damgård 结构的工作流程：
1. 将消息填充到固定长度的倍数
2. 将消息分成固定大小的块
3. 使用压缩函数逐块处理
4. 输出最终的哈希值

```
H0 = IV（初始值）
H1 = compress(H0, M1)
H2 = compress(H1, M2)
...
Hn = compress(Hn-1, Mn)

输出 = Hn
```

**安全性保证**

Merkle-Damgård 结构的安全性依赖于压缩函数的安全性：
- 如果压缩函数是抗碰撞的，则整体哈希函数也是抗碰撞的
- 压缩函数需要具有良好的扩散和混淆特性

**潜在问题**

Merkle-Damgård 结构的潜在问题：
- 长度扩展攻击：给定 H(M)，可以计算 H(M || X) 而不知道 M
- 多碰撞攻击：对于长消息，可能存在多碰撞

现代应用通过特定的使用模式（如 HMAC）避免这些问题。

### SHA-256 参数

SHA-256 的基本参数：

| 参数 | 值 |
|------|-----|
| 输出长度 | 256 位（32 字节） |
| 块大小 | 512 位（64 字节） |
| 字大小 | 32 位（4 字节） |
| 轮数 | 64 |
| 初始值 | 8 个 32 位常量 |

### 处理流程

SHA-256 的完整处理流程如下：

**第一步：消息填充**

将原始消息填充到 512 位（64 字节）的倍数。

填充规则：
1. 在消息末尾添加一个 1 位
2. 添加若干个 0 位，使消息长度为 448 mod 512
3. 在末尾附加 64 位（8 字节）的原始消息长度（大端序）

例如：
- 原始消息 "abc"：3 字节 = 24 位
- 填充后：24 位 + 1 位 + 423 位 + 64 位 = 512 位
- 输出：1 块

**第二步：初始化哈希值**

SHA-256 使用 8 个 32 位工作变量，初始值为特定的常量（由素数平方根的分数部分导出）：

```
H0 = 0x6a09e667  (√2 的分数部分)
H1 = 0xbb67ae85  (√3 的分数部分)
H2 = 0x3c6ef372  (√5 的分数部分)
H3 = 0xa54ff53a  (√7 的分数部分)
H4 = 0x510e527f  (√11 的分数部分)
H5 = 0x9b05688c  (√13 的分数部分)
H6 = 0x1f83d9ab  (√17 的分数部分)
H7 = 0x5be0cd19  (√19 的分数部分)
```

**第三步：消息扩展（消息调度）**

对于每个 512 位的消息块，将其扩展为 64 个 32 位字 W0, W1, ..., W63：

```
Wt = Mt                     (0 ≤ t ≤ 15)
Wt = σ1(Wt-2) + Wt-7 + σ0(Wt-15) + Wt-16  (16 ≤ t ≤ 63)
```

其中 σ0 和 σ1 是辅助函数：

```
σ0(x) = (x >>> 7) ⊕ (x >>> 18) ⊕ (x >> 3)
σ1(x) = (x >>> 17) ⊕ (x >>> 19) ⊕ (x >> 10)
```

>>> 表示循环右移，>> 表示逻辑右移。

**第四步：轮函数**

SHA-256 使用 64 轮迭代处理每个消息块。每轮更新 8 个工作变量 a, b, c, d, e, f, g, h。

轮函数结构：

```
T1 = h + Σ1(e) + Ch(e, f, g) + Kt + Wt
T2 = Σ0(a) + Maj(a, b, c)

h = g
g = f
f = e
e = d + T1
d = c
c = b
b = a
a = T1 + T2
```

辅助函数：

```
Ch(x, y, z) = (x ∧ y) ⊕ (¬x ∧ z)      // 选择函数
Maj(x, y, z) = (x ∧ y) ⊕ (x ∧ z) ⊕ (y ∧ z)  // 多数函数
Σ0(x) = (x >>> 2) ⊕ (x >>> 13) ⊕ (x >>> 22)  // 大 Σ0
Σ1(x) = (x >>> 6) ⊕ (x >>> 11) ⊕ (x >>> 25)  // 大 Σ1
```

**第五步：轮常量**

64 个轮常量 Kt 由前 64 个素数的立方根分数部分导出：

```
K0 = 0x428a2f98  (∛2 的分数部分)
K1 = 0x71374491  (∛3 的分数部分)
K2 = 0xb5c0fbcf  (∛5 的分数部分)
...
K63 = 0x7eb5c0e1
```

**第六步：更新哈希值**

处理完每个消息块后，将工作变量加到哈希值上：

```
H0 = H0 + a
H1 = H1 + b
H2 = H2 + c
H3 = H3 + d
H4 = H4 + e
H5 = H5 + f
H6 = H6 + g
H7 = H7 + h
```

**第七步：输出**

最终哈希值为 H0 || H1 || H2 || H3 || H4 || H5 || H6 || H7（共 256 位，32 字节）。

### 完整实现示例

```cpp
// SHA-256 实现（简化版）
std::array<std::uint8_t, 32> sha256(
    std::span<const std::uint8_t> message) {
    
    // 初始值
    std::array<std::uint32_t, 8> H = {
        0x6a09e667, 0xbb67ae85, 0x3c6ef372, 0xa54ff53a,
        0x510e527f, 0x9b05688c, 0x1f83d9ab, 0x5be0cd19
    };
    
    // 填充消息
    std::vector<std::uint8_t> padded = pad_message(message);
    
    // 处理每个块
    for (size_t block = 0; block < padded.size() / 64; ++block) {
        // 消息扩展
        std::array<std::uint32_t, 64> W = expand_message_block(
            padded.data() + block * 64);
        
        // 初始化工作变量
        std::array<std::uint32_t, 8> work = H;
        
        // 64 轮
        for (int t = 0; t < 64; ++t) {
            std::uint32_t T1 = work[7] + Sigma1(work[4]) + 
                               Ch(work[4], work[5], work[6]) + 
                               K[t] + W[t];
            std::uint32_t T2 = Sigma0(work[0]) + Maj(work[0], work[1], work[2]);
            
            work[7] = work[6];
            work[6] = work[5];
            work[5] = work[4];
            work[4] = work[3] + T1;
            work[3] = work[2];
            work[2] = work[1];
            work[1] = work[0];
            work[0] = T1 + T2;
        }
        
        // 更新哈希值
        for (int i = 0; i < 8; ++i) {
            H[i] += work[i];
        }
    }
    
    // 输出（大端序）
    std::array<std::uint8_t, 32> output;
    for (int i = 0; i < 8; ++i) {
        output[i * 4 + 0] = (H[i] >> 24) & 0xFF;
        output[i * 4 + 1] = (H[i] >> 16) & 0xFF;
        output[i * 4 + 2] = (H[i] >> 8) & 0xFF;
        output[i * 4 + 3] = H[i] & 0xFF;
    }
    
    return output;
}
```

```cpp
// SHA-256 辅助函数
inline std::uint32_t Ch(std::uint32_t x, std::uint32_t y, std::uint32_t z) {
    return (x & y) ^ (~x & z);
}

inline std::uint32_t Maj(std::uint32_t x, std::uint32_t y, std::uint32_t z) {
    return (x & y) ^ (x & z) ^ (y & z);
}

inline std::uint32_t Sigma0(std::uint32_t x) {
    return (x >> 2 | x << 30) ^ (x >> 13 | x << 19) ^ (x >> 22 | x << 10);
}

inline std::uint32_t Sigma1(std::uint32_t x) {
    return (x >> 6 | x << 26) ^ (x >> 11 | x << 21) ^ (x >> 25 | x << 7);
}

inline std::uint32_t sigma0(std::uint32_t x) {
    return (x >> 7 | x << 25) ^ (x >> 18 | x << 14) ^ (x >> 3);
}

inline std::uint32_t sigma1(std::uint32_t x) {
    return (x >> 17 | x << 15) ^ (x >> 19 | x << 13) ^ (x >> 10);
}

inline std::uint32_t ROTR(std::uint32_t x, int n) {
    return (x >> n) | (x << (32 - n));
}
```

### 安全性分析

**安全强度**

SHA-256 的安全强度：
- 碰撞抵抗：128 位（需要约 2^128 次操作找到碰撞）
- 原像抵抗：256 位（需要约 2^256 次操作找到原像）
- 第二原像抵抗：256 位

**安全性基础**

SHA-256 的安全性基于：
1. 压缩函数的设计：混淆和扩散特性
2. 消息扩展的线性性质：确保消息影响所有轮
3. 轮函数的非线性性质：防止逆向推导

**已知分析**

SHA-256 经过了广泛的密码学分析：
- 最佳已知碰撞攻击：约 2^126 次（仅略优于生日攻击）
- 最佳已知原像攻击：约 2^254 次
- 没有已知的实际攻击

SHA-256 是安全的，没有已知的有效攻击。

### 与其他算法的比较

| 算法 | 输出长度 | 安全强度 | 性能 |
|------|----------|----------|------|
| SHA-1 | 160 位 | 不安全（碰撞） | 高 |
| SHA-256 | 256 位 | 128 位 | 中 |
| SHA-384 | 384 位 | 192 位 | 中 |
| SHA-512 | 512 位 | 256 位 | 高（64位） |
| SHA3-256 | 256 位 | 128 位 | 中 |

SHA-256 是安全性和性能的良好平衡点。

Reality 协议使用 SHA-256 作为 HKDF 的底层哈希函数。

**HKDF-SHA256**

```cpp
// Reality 使用 HKDF-SHA256 派生密钥
session_keys derive_reality_keys(
    std::span<const std::uint8_t, 32> shared_secret,
    std::span<const std::uint8_t> handshake_hash) {
    
    session_keys keys;
    
    // HKDF-Extract 使用 HMAC-SHA256
    auto early_secret = hkdf_extract_sha256({}, {});
    auto handshake_secret = hkdf_extract_sha256(early_secret, shared_secret);
    
    // HKDF-Expand-Label
    keys.client_key = hkdf_expand_label_sha256(handshake_secret, "key", handshake_hash, 16);
    keys.server_key = hkdf_expand_label_sha256(handshake_secret, "key", handshake_hash, 16);
    
    return keys;
}

// HMAC-SHA256 实现
std::array<std::uint8_t, 32> hmac_sha256(
    std::span<const std::uint8_t> key,
    std::span<const std::uint8_t> message) {
    
    const size_t block_size = 64;
    std::vector<std::uint8_t> padded_key(block_size, 0);
    
    if (key.size() > block_size) {
        auto key_hash = sha256(key);
        std::copy(key_hash.begin(), key_hash.end(), padded_key.begin());
    } else {
        std::copy(key.begin(), key.end(), padded_key.begin());
    }
    
    // 内部哈希
    std::vector<std::uint8_t> inner_input(block_size);
    for (size_t i = 0; i < block_size; ++i) {
        inner_input[i] = padded_key[i] ^ 0x36;
    }
    inner_input.insert(inner_input.end(), message.begin(), message.end());
    auto inner_hash = sha256(inner_input);
    
    // 外部哈希
    std::vector<std::uint8_t> outer_input(block_size);
    for (size_t i = 0; i < block_size; ++i) {
        outer_input[i] = padded_key[i] ^ 0x5C;
    }
    outer_input.insert(outer_input.end(), inner_hash.begin(), inner_hash.end());
    
    return sha256(outer_input);
}
```

```cpp
// 增量 SHA-256
class incremental_sha256 {
public:
    incremental_sha256() {
        SHA256_Init(&ctx_);
    }
    
    void update(std::span<const std::uint8_t> data) {
        SHA256_Update(&ctx_, data.data(), data.size());
    }
    
    std::array<std::uint8_t, 32> finalize() {
        std::array<std::uint8_t, 32> output;
        SHA256_Final(output.data(), &ctx_);

    }

};
```

## 安全考量

### 已知安全问题

**长度扩展攻击**

SHA-256 基于 Merkle-Damgård 结构，易受长度扩展攻击：
- 给定 H(M) 和 len(M)，可以计算 H(M || padding || X)
- 不需要知道 M 的内容

防御措施：
- 使用 HMAC 构造
- 使用 SHA-3（Keccak）
- 在协议设计中考虑此问题

**多碰撞**

理论上存在多碰撞问题：
- 对于 n 消息碰撞，可以找到更多碰撞
- 对实际应用影响有限

### 安全建议

1. **正确使用**
   - 使用 HMAC-SHA256 进行消息认证
   - 使用 HKDF-SHA256 进行密钥派生
   - 避免直接使用 SHA-256 进行认证

2. **输出处理**
   - 32 字节输出已足够安全
   - 截断输出会降低安全性
   - 使用完整输出进行关键操作

3. **性能考虑**
   - 使用硬件加速（SHA-NI）
   - 对于大数据使用增量哈希
   - 避免不必要的哈希计算

### 安全边界

| 攻击类型 | 安全强度 | 说明 |
|----------|----------|------|
| 碰撞攻击 | 128 位 | 生日攻击界限 |
| 原像攻击 | 256 位 | 理论值 |
| 第二原像 | 256 位 | 理论值 |

## 最佳实践

### HMAC 构造

**使用 HMAC-SHA256**

```cpp
// HMAC-SHA256 标准实现
std::array<std::uint8_t, 32> hmac_sha256(
    std::span<const std::uint8_t> key,
    std::span<const std::uint8_t> message) {
    
    // 使用 OpenSSL/BoringSSL 实现
    std::array<std::uint8_t, 32> result;
    HMAC(EVP_sha256(), key.data(), key.size(), 
         message.data(), message.size(),
         result.data(), nullptr);
    
    return result;
}
```

### 密钥派生

**使用 HKDF-SHA256**

```cpp
// HKDF-SHA256 密钥派生
std::vector<std::uint8_t> hkdf_sha256(
    std::span<const std::uint8_t> salt,
    std::span<const std::uint8_t> ikm,
    std::span<const std::uint8_t> info,
    size_t length) {
    
    // Extract
    auto prk = hmac_sha256(salt, ikm);
    
    // Expand
    std::vector<std::uint8_t> okm;
    std::vector<std::uint8_t> t_prev;
    
    size_t iterations = (length + 31) / 32;
    
    for (size_t i = 1; i <= iterations; ++i) {
        std::vector<std::uint8_t> input;
        input.insert(input.end(), t_prev.begin(), t_prev.end());
        input.insert(input.end(), info.begin(), info.end());
        input.push_back(static_cast<std::uint8_t>(i));
        
        t_prev = hmac_sha256(prk, input);
        
        size_t bytes_to_add = (i == iterations) 
            ? (length - okm.size()) : 32;
        okm.insert(okm.end(), t_prev.begin(), t_prev.begin() + bytes_to_add);
    }
    
    return okm;
}
```

**文件完整性校验**

```cpp
// 文件完整性校验
class file_integrity_checker {
public:
    std::array<std::uint8_t, 32> compute_file_hash(
        const std::string& filepath) {
        
        incremental_sha256 hasher;
        
        std::ifstream file(filepath, std::ios::binary);
        std::array<std::uint8_t, 4096> buffer;
        
        while (file.read(reinterpret_cast<char*>(buffer.data()), buffer.size())) {
            hasher.update(buffer);
        }
        
        if (file.gcount() > 0) {
            hasher.update(std::span(buffer.data(), file.gcount()));
        }
        
        return hasher.finalize();
    }
    
    bool verify_file(
        const std::string& filepath,
        std::span<const std::uint8_t, 32> expected_hash) {
        
        auto computed = compute_file_hash(filepath);
        return constant_time_compare(computed, expected_hash);
    }
};
```

### 恒定时间比较

```cpp
// 恒定时间哈希比较
bool constant_time_compare(
    std::span<const std::uint8_t> a,
    std::span<const std::uint8_t> b) {
    
    if (a.size() != b.size()) {
        return false;
    }
    
    std::uint8_t result = 0;
    for (size_t i = 0; i < a.size(); ++i) {
        result |= a[i] ^ b[i];
    }
    
    return result == 0;
}
```

## 常见问题

### Q1: SHA-256 与 SHA-1 有什么区别？

主要区别：
- 输出长度：SHA-256 为 256 位，SHA-1 为 160 位
- 安全性：SHA-256 安全，SHA-1 有碰撞攻击
- 轮数：SHA-256 为 64 轮，SHA-1 为 80 轮
- 结构：SHA-256 有更多的工作变量和辅助函数

SHA-1 已被弃用，新系统应使用 SHA-256。

### Q2: SHA-256 的输出长度足够吗？

是的。256 位输出提供：
- 128 位碰撞抵抗（生日攻击）
- 256 位原像抵抗

对于绝大多数应用，这已经足够安全。截断输出会降低安全性。

### Q3: SHA-256 可以用作 MAC 吗？

不能直接用作 MAC。SHA-256 是哈希函数，没有密钥输入。应使用：
- HMAC-SHA256：标准的 MAC 构造
- 使用密钥派生后再加密

直接将密钥与消息拼接后哈希是不安全的。

### Q4: SHA-256 与 SHA3-256 如何选择？

比较：
- SHA-256：广泛支持，硬件加速，NIST 标准
- SHA3-256：更新设计，无长度扩展问题，Keccak 结构

SHA-256 是大多数场景的默认选择。SHA3-256 适用于：
- 需要避免长度扩展攻击的场景
- 需要多样化哈希算法的场景

### Q5: SHA-256 在 TLS 1.3 中的作用是什么？

TLS 1.3 使用 SHA-256：
- HKDF-SHA256：所有密钥派生
- HMAC-SHA256：记录层 MAC
- 握手哈希： transcripts 计算

SHA-256 是 TLS 1.3 的默认哈希函数。

### Q6: SHA-256 可以截断吗？

可以截断，但会降低安全性：
- 截断到 128 位：64 位碰撞抵抗（不推荐）
- 截断到 192 位：96 位碰撞抵抗（可接受）
- 截断到 224 位：112 位碰撞抵抗（可接受）

建议使用完整的 256 位输出。

### Q7: SHA-256 有硬件加速吗？

是的：
- Intel SHA-NI：专门的 SHA 指令
- ARMv8 SHA 扩展：类似功能
- 性能提升：约 3-10 倍

现代处理器通常支持 SHA 加速。

### Q8: SHA-256 的性能如何？

典型性能（单线程）：
- 纯软件：约 200-300 MB/s
- SHA-NI 加速：约 500-1000 MB/s

对于大多数应用，SHA-256 性能足够。

### Q9: SHA-256 与比特币的关系？

比特币使用 SHA-256：
- 工作量证明：SHA256(SHA256(block_header))
- 交易哈希：SHA256(SHA256(transaction))
- 地址生成：从公钥派生

比特币的双重哈希增加安全性。

### Q10: SHA-256 可以用于密码哈希吗？

不建议直接使用。密码哈希需要：
- 慢哈希：防止暴力破解
- 盐值：防止彩虹表攻击
- 内存密集：防止硬件加速攻击

应使用 PBKDF2、Argon2 或 bcrypt 进行密码哈希
