# 代理检测原理

**类别**: 网络

## 概述

代理检测是识别代理服务器和代理流量的技术。理解检测原理有助于设计不易被识别的代理方案，提升代理的隐蔽性。

### 检测维度

代理检测的主要维度：

| 维度 | 说明 | 检测方法 |
|------|------|----------|
| **TLS 指纹** | ClientHello 特征 | JA3 签名 |
| **协议特征** | 代理协议模式 | DPI 匹配 |
| **行为特征** | 连接行为异常 | 统计分析 |
| **流量特征** | 数据传输模式 | 流量分析 |
| **主动探测** | 验证服务响应 | 探测请求 |

### 检测层级

代理检测在不同层级的检测：

```
检测层级：

TLS 层检测：
┌─────────────────────────────────────────────┐
│ TLS ClientHello 分析                         │
│ - JA3 指纹签名                               │
│ - SNI 域名检测                               │
│ - Certificate 分析                           │
└─────────────────────────────────────────────┘

应用层检测：
┌─────────────────────────────────────────────┐
│ 协议特征匹配                                  │
│ - Shadowsocks 特征                           │
│ - Trojan 特征                                │
│ - V2Ray 特征                                 │
└─────────────────────────────────────────────┘

流量层检测：
┌─────────────────────────────────────────────┐
│ 流量统计分析                                  │
│ - 包大小分布                                  │
│ - 时间间隔                                    │
│ - 速率变化                                    │
└─────────────────────────────────────────────┘

主动探测：
┌─────────────────────────────────────────────┐
│ 发送探测请求                                  │
│ - TLS 连接尝试                                │
│ - 协议探测包                                  │
│ - 响应分析                                    │
└─────────────────────────────────────────────┘
```

## TLS 指纹检测

### JA3 签名

JA3 TLS 指纹方法：

```
JA3 签名原理：

ClientHello 包含：
- Version: TLS 版本
- Cipher Suites: 加密套件列表
- Extensions: 扩展列表
- Supported Groups: 支持的曲线
- Signature Algorithms: 签名算法

JA3 计算：
MD5(Version + CipherSuites + Extensions + SupportedGroups + SignatureAlgorithms)

示例：
ClientHello: TLS 1.2, ciphers=[49195,49199,...], extensions=[0,10,11,...]
JA3 = "769,49195-49199-...,0-10-11-...,..."

指纹数据库：
- 浏览器指纹已知（Chrome, Firefox, Safari）
- 代理客户端指纹异常
- 检测非标准指纹
```

### 指纹异常检测

非标准指纹的检测：

```
指纹异常检测：

正常指纹示例：
| 客户端 | JA3 值 |
|--------|--------|
| Chrome | "769,49195-49199-...,..." |
| Firefox | "769,49195-49196-...,..." |
| Safari | "769,49195-...,..." |

异常指纹：
- 代理客户端（Go, Python）有不同指纹
- 指纹不在已知数据库
- 扩展顺序异常

检测逻辑：
1. 计算 JA3 签名
2. 查询已知指纹库
3. 未匹配 → 标记可疑
4. 可疑流量 → 触发进一步检测
```

详见 [[ref/anti-censorship/tls-fingerprint|TLS 指纹]]。

### 指纹伪装

TLS 指纹伪装方法：

```
指纹伪装方法：

方法 1: 使用标准 TLS 库
- 使用浏览器 TLS 库
- 模拟标准 ClientHello

方法 2: Reality 方案
- ClientHello 来自真实客户端
- 服务端看到真实指纹
- 无代理特征暴露

方法 3: 自定义配置
- 配置 cipher suites 顺序
- 配置 extensions 顺序
- 模拟浏览器指纹

挑战：
- 库版本影响指纹
- 需要持续更新
- Reality 方案更可靠
```

## 协议特征检测

### Shadowsocks 检测

Shadowsocks 协议特征：

```
Shadowsocks 检测特征：

早期版本：
- 固定长度握手包
- 特定加密特征
- 流量统计特征

检测方法：
- 流量统计分析
- 固定包长匹配
- 加密模式识别

绕过：
- SS2022 新协议
- 流量混淆插件
- Reality 包装

SS2022 特性：
- 随机 padding
- 更自然的流量模式
- 但仍可能被检测
```

### Trojan 检测

Trojan 协议特征：

```
Trojan 检测特征：

协议结构：
TLS + Trojan 认证 + 数据

特征：
- TLS 后的固定格式认证
- 认证字段（SHA-224）
- 特定端口行为

检测方法：
- TLS 后数据分析
- 流量统计特征
- 端口行为分析

绕过：
- Reality（更隐蔽）
- 隐藏 Trojan 认证
- 流量混淆
```

详见 [[ref/anti-censorship/dpi|DPI]]。

## 行为特征检测

### 连接行为分析

连接行为的异常检测：

```
连接行为分析：

正常行为：
- 短连接（HTTP）或长连接（HTTPS）
- 连接频率适中
- 连接目标多样

异常行为：
- 高频连接
- 长时间连接（代理转发）
- 连接目标集中
- 连接模式固定

检测维度：
| 维度 | 正常 | 异常 |
|------|------|------|
| 连接时长 | 短或合理长 | 非常长 |
| 连接频率 | 适中 | 高频 |
| 目标分布 | 分散 | 集中 |
| 端口分布 | 多端口 | 单端口 |

绕过：
- 模拟正常连接模式
- 连接池复用减少连接数
- 分散流量
```

### 端口行为分析

端口使用的异常检测：

```
端口行为分析：

正常端口：
- HTTP: 80
- HTTPS: 443
- 其他服务端口

异常端口：
- 非标准端口代理
- 高端口频繁使用
- 端口扫描行为

检测方法：
- 监控端口使用频率
- 检测非标准端口
- 分析端口分布

绕过：
- 使用标准端口（443）
- 模拟 HTTPS 流量
- Reality 使用标准 TLS
```

## 流量特征检测

### 包大小分析

数据包大小的统计特征：

```
包大小分析：

正常 HTTPS：
- 包大小分布多样
- 大包（传输数据）
- 小包（控制、ACK）

代理流量：
- 包大小分布异常
- 固定大小包
- 均匀分布

检测方法：
- 包大小分布统计
- 固定长度检测
- 均匀性分析

绕过：
- 添加随机 padding
- 模拟正常包大小分布
- Reality 使用真实 TLS 流量
```

### 时序分析

数据包时间间隔分析：

```
时序分析：

正常流量：
- 时间间隔变化
- 有规律（如网页加载）
- 交互式（请求-响应）

代理流量：
- 时间间隔均匀
- 持续传输
- 无明显交互模式

检测方法：
- 时间间隔统计
- 时序模式识别
- 机器学习模型

绕过：
- 添加随机延迟
- 模拟交互模式
- 分散传输
```

详见 [[ref/anti-censorship/traffic-analysis|流量分析]]。

## 主动探测

### 探测方法

主动探测可疑服务：

```
主动探测方法：

探测类型：
1. TLS 探测
   - 建立 TLS 连接
   - 发送 HTTP 请求
   - 分析响应

2. 协议探测
   - 发送代理协议数据
   - 检测代理响应
   - 确认代理类型

3. 行为探测
   - 模拟客户端行为
   - 检测服务响应
   - 区分代理和正常服务

探测特征：
- 特定 IP 来源
- 特定 TLS 指纹
- 特定探测模式
```

### 探测识别

识别探测请求：

```
探测识别方法：

探测特征：
- TLS 指纹异常（非浏览器）
- 请求内容异常
- 来源 IP 异常

识别策略：
1. 检测 TLS 指纹
   - 探测者使用特殊 TLS 库
   - 指纹与浏览器不同

2. 检测请求模式
   - 探测请求特定内容
   - 正常客户端不同

3. 检测来源
   - 探测 IP 特定范围
   - 黑名单 IP
```

### 探测绕过

绕过主动探测的方法：

```
探测绕过策略：

策略 1: Reality
- 验证客户端是真实用户
- 探测者无法获得代理响应
- 探测得到的是真实网站响应

策略 2: 伪装响应
- 探测时返回正常 HTTP 响应
- 不暴露代理特征

策略 3: 检测并拒绝
- 检测探测特征
- 拒绝探测连接
- 不响应探测请求

Reality 优势：
- 探测者看到真实网站
- 无法区分代理和真实网站
- 最隐蔽的方案
```

详见 [[core/stealth/reality/handshake|Reality]]。

方案：
- Reality: 最隐蔽的 TLS 伪装

设计目标：
- 绕过 TLS 指纹检测

Reality 核心特性：
