# X25519

**类别**: 密码学

## 概述

X25519 是基于 Curve25519 椭圆曲线的 Diffie-Hellman 密钥交换协议，由 Daniel J. Bernstein 在 2006 年设计并发布。X25519 是现代密码学中最重要的密钥交换算法之一，广泛用于 TLS 1.3、SSH、Signal 协议、WireGuard VPN 等安全通信系统。

Curve25519 是一条精心设计的椭圆曲线，方程为 y² = x³ + 486662x² + x (mod p)，其中 p = 2²⁵⁵ - 19。这个特定选择使得曲线在数学和工程上都具有优越的属性：素数域接近 2 的幂次便于高效计算，曲线参数经过优化避免各种已知攻击，并且所有计算都可以在恒定时间内完成。

X25519 的核心优势在于其安全性、性能和实现简单性。与传统的 NIST 曲线（如 P-256）相比，X25519 提供更强的安全保证（128 位安全强度），在软件实现中性能更高（约 3-10 倍），并且所有操作天然是恒定时间的，有效防止时序攻击。这些优势使得 X25519 成为现代密码学应用的首选密钥交换算法。

X25519 使用 Montgomery 形式的椭圆曲线，这种形式支持高效的 x-only 点运算。通过 ladder 算法，X25519 可以在不计算 y 坐标的情况下完成点乘运算，这不仅简化了实现，还确保了恒定时间执行。这种设计巧妙地避免了传统 Weierstrass 形式曲线需要复杂处理的边界条件。

X25519 的密钥格式非常简单：32 字节的公钥和 32 字节的私钥。私钥在生成时会经过特定的位处理（clamping），清除低位、设置高位，这使得所有私钥都在安全的范围内，避免了小私钥攻击。公钥直接就是点乘结果，不需要额外的编码或压缩。

在 TLS 1.3 中，X25519 是强制支持的密钥交换曲线。TLS 1.3 规定了三种密钥交换组：x25519（强制）、secp256r1（可选）和 x448（可选）。X25519 作为默认和强制支持的曲线，确保了所有 TLS 1.3 实现都具备这一现代化的密钥交换能力。

WireGuard VPN 协议使用 X25519 作为其唯一的密钥交换算法。WireGuard 的设计哲学强调简单和高效，X25519 正好满足这些要求。WireGuard 的握手协议使用 Noise Framework，其中 X25519 用于长期密钥和临时密钥的交换。

Signal 协议（用于 WhatsApp、Signal 等安全聊天应用）使用 X25519 进行密钥协商。Signal 的 Double Ratchet 算法使用 X25519 进行每次消息的密钥派生，确保了前向保密和未来保密。

X25519 的安全性经过了广泛的密码学分析。曲线参数的选择经过了精心设计，避免了已知的椭圆曲线攻击，包括小阶攻击、扭曲曲线攻击、无效曲线攻击等。X25519 的 cofactor 为 8，这意味着存在小子群，但 Bernstein 的设计通过私钥 clamping 确保点乘结果始终在主子群中。

在现代处理器上，X25519 的性能非常出色。单次点乘操作可以在几十微秒内完成，这使得 X25519 适合高频密钥交换场景。在 ARM Cortex-M0 等嵌入式处理器上，X25519 也能在毫秒级完成，这使得它适用于 IoT 设备。

X25519 的实现相对简单，只需要几千行代码就能完成。这与需要复杂异常处理的 Weierstrass 曲线实现相比，大大减少了实现错误的可能性。许多安全漏洞（如 Apple 的 goto fail、OpenSSL 的 Heartbleed）都与复杂的边界处理有关，而 X25519 的设计天然避免了这些问题。

在代理服务器和网络协议实现中，X25519 是密钥交换的核心算法。Prism 在 Reality 协议中使用 X25519 进行客户端和服务端的密钥协商，然后通过 HKDF 派生出会话加密密钥。X25519 提供了高效、安全的密钥交换基础。

总的来说，X25519 代表了现代椭圆曲线密码学的最佳实践。它的设计体现了"安全工程"的理念：通过数学上的精心选择和工程上的简化设计，在安全性、性能和实现简单性之间取得了最佳平衡。理解 X25519 的工作原理对于正确实现和使用现代安全协议至关重要。

## 算法原理

### Curve25519 椭圆曲线

Curve25519 是一条 Montgomery 形式的椭圆曲线，定义如下：

**曲线方程**

```
y² = x³ + 486662x² + x (mod p)
```

其中 p = 2²⁵⁵ - 19 是一个接近 2²⁵⁵ 的素数。

**参数选择的原因**

1. **素数 p = 2²⁵⁵ - 19**
   - 接近 2 的幂次，便于高效计算
   - 19 是小素数，模约简简单
   - 255 位，便于 32 字节表示

2. **曲线参数 A = 486662**
   - 避免小阶点攻击
   - 曲线阶接近素数（cofactor = 8）
   - 与扭曲曲线区分良好

3. **基点**
   - Montgomery ladder 使用固定的基点
   - 基点 u = 9，这是一个小整数
   - 基点阶为 (p+1-A)/8 = 素数

**Montgomery 形式的优势**

Montgomery 形式的椭圆曲线有以下优势：
1. **x-only 运算**：只需要 x 坐标，不需要 y 坐标
2. **恒定时间 ladder**：算法天然恒定时间
3. **无异常处理**：不存在需要特殊处理的边界条件
4. **高效实现**：每比特只需约 10 次域乘法

### X25519 密钥交换协议

X25519 使用 Montgomery ladder 算法进行点乘运算。

**密钥生成**

```cpp
// X25519 私钥生成
std::array<std::uint8_t, 32> generate_x25519_private_key() {
    std::array<std::uint8_t, 32> private_key;
    RAND_bytes(private_key.data(), 32);
    
    // Clamping（位处理）
    private_key[0] &= 0xF8;    // 清除最低 3 位
    private_key[31] &= 0x7F;   // 清除最高位
    private_key[31] |= 0x40;   // 设置次高位
    
    return private_key;
}
```

Clamping 的作用：
- 清除最低 3 位：确保私钥乘以 cofactor 后为 8 的倍数
- 设置最高位为 0、次高位为 1：确保私钥在正确范围内

**公钥计算**

公钥通过点乘计算：PublicKey = PrivateKey * BasePoint

基点 u = 9（表示为 32 字节）。

```cpp
// X25519 公钥计算
std::array<std::uint8_t, 32> compute_x25519_public_key(
    std::span<const std::uint8_t, 32> private_key) {
    
    // Montgomery ladder 点乘
    return montgomery_ladder(private_key, base_point_u);
}
```

**共享密钥计算**

双方各自计算共享密钥：
- Alice: SharedSecret = AlicePrivateKey * BobPublicKey
- Bob: SharedSecret = BobPrivateKey * AlicePublicKey

由于椭圆曲线的性质，两个结果相等。

```cpp
// X25519 共享密钥计算
std::array<std::uint8_t, 32> compute_x25519_shared_secret(
    std::span<const std::uint8_t, 32> private_key,
    std::span<const std::uint8_t, 32> peer_public_key) {
    
    return montgomery_ladder(private_key, peer_public_key);
}
```

### Montgomery Ladder 算法

Montgomery ladder 是 X25519 的核心算法，实现了恒定时间的点乘。

**算法描述**

输入：点 u（255 位整数）、标量 k（私钥）
输出：k * u（点乘结果）

```
算法 MontgomeryLadder(u, k):
    // 初始化：P0 = 0（无穷远点），P1 = u
    x_1 = u
    x_2 = 1
    z_2 = 0
    x_3 = u
    z_3 = 1
    
    // 从高位到低位处理每个比特
    swap = 0
    for t = 254 downto 0:
        k_t = (k >> t) & 1
        swap ^= k_t
        
        // 条件交换（恒定时间）
        cswap(swap, x_2, x_3)
        cswap(swap, z_2, z_3)
        swap = k_t
        
        // Montgomery double-and-add
        A = x_2 + z_2
        AA = A²
        B = x_2 - z_2
        BB = B²
        E = AA - BB
        C = x_3 + z_3
        D = x_3 - z_3
        DA = D * A
        CB = C * B
        x_3 = (DA + CB)²
        z_3 = x_1 * (DA - CB)²
        x_2 = AA * BB
        z_2 = E * (AA + a24 * E)
    
    // 最后一次交换
    cswap(swap, x_2, x_3)
    cswap(swap, z_2, z_3)
    
    // 返回 x_2 / z_2
    return x_2 * (z_2^(p-2)) mod p
```

其中 a24 = (A + 2) / 4 = 121666，是曲线参数的变形。

**恒定时间保证**

Montgomery ladder 的恒定时间特性来自：
1. 每个比特执行相同的操作序列
2. 条件交换通过 XOR 实现（cswap）
3. 不依赖输入值的分支

**条件交换实现**

```cpp
// 恒定时间条件交换
void cswap(std::uint32_t swap, std::uint32_t& a, std::uint32_t& b) {
    std::uint32_t dummy = swap * (a ^ b);
    a ^= dummy;
    b ^= dummy;
}
```

当 swap = 0，a 和 b 不交换；当 swap = 1，a 和 b 交换。

### 安全性分析

**安全强度**

X25519 提供 128 位的安全强度。这意味着：
- 破解密钥交换需要约 2¹²⁸ 次计算
- 与 AES-128 的安全强度相当
- 远高于实际安全需求

**已知攻击的防御**

1. **小阶攻击**：私钥 clamping 确保结果在主子群
2. **扭曲曲线攻击**：曲线参数选择与扭曲曲线区分良好
3. **无效曲线攻击**：Montgomery 形式天然防御
4. **时序攻击**：恒定时间 ladder 算法

**Cofactor 处理**

Curve25519 的 cofactor 为 8，意味着存在 8 个元素的子群。私钥 clamping 确保：
- 私钥最低 3 位为 0，私钥乘以 cofactor 后为 8 的倍数
- 点乘结果始终在主子群（阶为素数）
- 小子群元素被 cofactor 吞噬

### 实现细节

**完整点乘实现（简化版）**

```cpp
std::array<std::uint8_t, 32> x25519_scalar_multiply(
    std::span<const std::uint8_t, 32> scalar,
    std::span<const std::uint8_t, 32> u) {
    
    // 解析输入为域元素
    std::array<std::uint32_t, 8> k = decode_scalar(scalar);
    std::array<std::uint32_t, 8> x_1 = decode_u(u);
    
    // Montgomery ladder
    std::array<std::uint32_t, 8> x_2 = {1, 0, 0, 0, 0, 0, 0, 0};
    std::array<std::uint32_t, 8> z_2 = {0, 0, 0, 0, 0, 0, 0, 0};
    std::array<std::uint32_t, 8> x_3 = x_1;
    std::array<std::uint32_t, 8> z_3 = {1, 0, 0, 0, 0, 0, 0, 0};
    
    std::uint32_t swap = 0;
    
    for (int t = 254; t >= 0; --t) {
        std::uint32_t k_t = (k[0] >> t) & 1;  // 取比特
        
        swap ^= k_t;
        cswap_arrays(swap, x_2, x_3);
        cswap_arrays(swap, z_2, z_3);
        swap = k_t;
        
        // Double and add 操作
        montgomery_step(x_1, x_2, z_2, x_3, z_3);
    }
    
    cswap_arrays(swap, x_2, x_3);
    cswap_arrays(swap, z_2, z_3);
    
    // 计算 x_2 / z_2
    std::array<std::uint32_t, 8> result = field_mul(x_2, field_inv(z_2));
    
    // 编码输出
    return encode_u(result);
}
```

```cpp
    std::span<const std::uint8_t> auth_token) {
```

## 安全考量

### 已知安全问题

**小子群攻击**

Curve25519 的 cofactor 为 8，存在小子群元素。如果对手使用小子群元素作为公钥，可能导致：
- 共享密钥泄露
- 子群限制攻击

防御措施：私钥 clamping 确保点乘结果在主子群。

**扭曲曲线攻击**

对手可能使用不在 Curve25519 上的"扭曲曲线"点。这可能导致：
- 共享密钥泄露不同值
- 验证失败时泄露信息

防御措施：Curve25519 的扭曲曲线与原曲线区分良好，clamping 进一步保护。

**公钥验证**

传统 ECDH 需要验证公钥是否在曲线上。X25519 通过以下方式避免显式验证：
- Montgomery ladder 可以处理任意输入
- clamping 确保私钥安全
- 低阶点产生确定性输出（可用于检测）

### 安全建议

1. **密钥管理**
   - 使用密码学安全的随机源生成私钥
   - 正确执行 clamping
   - 定期轮换密钥

2. **共享密钥处理**
   - 共享密钥不应直接用作加密密钥
   - 使用 HKDF 派生会话密钥
   - 使用后安全清理

3. **前向保密**
   - 使用临时密钥（每次会话新的密钥对）
   - 会话结束后删除临时私钥

4. **实现验证**
   - 使用经过验证的加密库实现
   - 测试恒定时间执行
   - 测试特殊输入处理

### 安全边界

| 属性 | 值 | 说明 |
|------|-----|------|
| 安全强度 | 128 位 | 等效于 AES-128 |
| 私钥长度 | 256 位 | clamping 后有效 252 位 |
| 公钥长度 | 256 位 | x 坐标编码 |
| Cofactor | 8 | 小子群存在 |

### 密钥生成

**安全的私钥生成**

```cpp
std::array<std::uint8_t, 32> generate_safe_x25519_private_key() {
    std::array<std::uint8_t, 32> private_key;
    
    // 使用硬件随机源
    if (RAND_bytes(private_key.data(), 32) != 1) {
        throw std::runtime_error("Random generation failed");
    }
    
    // Clamping
    private_key[0] &= 248;    // 0xF8
    private_key[31] &= 127;   // 0x7F
    private_key[31] |= 64;    // 0x40
    
    return private_key;
}
```

```cpp
class ephemeral_key_exchange {
public:
    ephemeral_key_exchange() {
        ephemeral_private_ = generate_safe_x25519_private_key();
        ephemeral_public_ = x25519_scalar_multiply(ephemeral_private_, base_point);

        std::span<const std::uint8_t, 32> peer_ephemeral_public) 
        -> negotiated_session {

        auto ephemeral_secret = x25519_scalar_multiply(
            ephemeral_private_, peer_ephemeral_public);
        
        // 如果有长期密钥，计算长期共享密钥
        if (has_long_term_key_) {
            auto long_term_secret = x25519_scalar_multiply(
                long_term_private_, peer_long_term_public_);
            
            // 组合两个共享密钥
            ephemeral_secret = combine_secrets(ephemeral_secret, long_term_secret);
        }
        
        // 使用 HKDF 派生会话密钥
        return derive_session_keys(ephemeral_secret);
    }
    
    ~ephemeral_key_exchange() {
        // 安全清理临时私钥
        OPENSSL_cleanse(ephemeral_private_.data(), 32);
    }
    
private:
    std::array<std::uint8_t, 32> ephemeral_private_;
    std::array<std::uint8_t, 32> ephemeral_public_;
    bool has_long_term_key_ = false;
};
```

```cpp
session_keys derive_session_keys(
    std::span<const std::uint8_t, 32> shared_secret) {
    
    session_keys keys;
    
    // 使用 HKDF 派生多个密钥
    auto master_secret = hkdf_extract({}, shared_secret);
    
    // 派生加密密钥
    auto expand_1 = hkdf_expand(master_secret, "encryption", 32);
    keys.encryption_key = hkdf_expand(expand_1, "key", 16);
    keys.encryption_iv = hkdf_expand(expand_1, "iv", 12);
    
    // 派生 MAC 密钥
    keys.mac_key = hkdf_expand(master_secret, "mac", 32);
    
    return keys;
}
```

### 错误处理

```cpp
auto safe_key_exchange(
    std::span<const std::uint8_t, 32> peer_public_key) 
    -> std::optional<std::array<std::uint8_t, 32>> {
    
    // 检查公钥是否为零或全 255
    bool is_zero = all_bytes_equal(peer_public_key, 0);
    bool is_all_255 = all_bytes_equal(peer_public_key, 255);
    
    if (is_zero || is_all_255) {
        // 低阶点，返回错误
        return std::nullopt;
    }
    
    // 计算共享密钥
    auto shared = x25519_scalar_multiply(private_key_, peer_public_key);
    
    // 检查共享密钥是否为零
    if (all_bytes_equal(shared, 0)) {
        return std::nullopt;
    }
    
    return shared;
}
```

## 常见问题

### Q1: X25519 与 P-256 如何选择？

X25519 相比 P-256 的优势：
- 更强的安全性保证（无 NSA 参与，参数公开选择）
- 更高的软件性能（约 3-10 倍）
- 恒定时间执行天然保证
- 更简单的实现（无异常处理）

建议新项目优先使用 X25519。

### Q2: X25519 的安全强度是多少？

X25519 提供 128 位安全强度。这意味着破解密钥交换需要约 2¹²⁸ 次操作，与 AES-128 的安全强度相当。

### Q3: 为什么私钥需要 clamping？

Clamping 的目的：
- 清除最低 3 位：确保私钥乘以 cofactor 后为 8 的倍数，避免小子群攻击
- 设置高位：确保私钥在正确范围内，避免小私钥攻击

这使得所有私钥都在安全范围内。

### Q4: X25519 需要公钥验证吗？

传统 ECDH 需要验证公钥在曲线上，但 X25519 不需要显式验证：
- Montgomery ladder 可以处理任意输入
- clamping 保护私钥安全
- 低阶点产生确定性输出（可以检测但不必须）

某些实现仍会检测零或全 255 公钥以提前失败。

### Q5: 共享密钥可以直接用作加密密钥吗？

不建议。共享密钥应该：
- 使用 HKDF 派生具体的加密密钥
- 包含协议上下文信息
- 确保密钥分离

直接使用共享密钥可能导致密钥误用。

### Q6: X25519 支持前向保密吗？

是的。X25519 可以用于 ECDHE（临时 ECDH）：
- 每次会话生成新的临时密钥对
- 会话结束后删除临时私钥
- 即使长期私钥泄露，历史会话仍安全

TLS 1.3 强制使用 ECDHE。

### Q7: X25519 在 ARM 上性能如何？

X25519 在 ARM 平台上性能良好：
- ARM Cortex-A72：约 1-2 ms 单次点乘
- ARM Cortex-M4：约 10-20 ms 单次点乘
- 比 P-256 在纯软件实现上快约 3-10 倍

### Q8: X25519 与 X448 有什么区别？

X448 是 X25519 的变体，使用 Curve448：
- 更长密钥：56 字节（448 位）
- 更高安全强度：224 位
- 更慢：约 2-3 倍

X25519 是默认选择，X448 用于更高安全需求。

### Q9: Curve25519 为什么选择 Montgomery 形式？

Montgomery 形式的优势：
- 只需要 x 坐标，简化实现
- Montgomery ladder 天然恒定时间
- 无需处理边界异常
- 高效的 double-and-add

这与 Weierstrass 形式相比大大简化了实现。

### Q10: Reality 协议为什么使用 X25519？

Reality 使用 X25519 的原因：
- 高安全性：128 位安全强度
- 高性能：快速密钥交换
- 恒定时间：防止时序攻击
- 简单实现：减少漏洞可能性
- 广泛支持：现代加密库都支持
