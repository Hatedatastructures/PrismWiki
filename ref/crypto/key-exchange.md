# 密钥交换原理

**类别**: 密码学

## 概述

密钥交换（Key Exchange）是在不安全信道上协商共享密钥的技术。现代协议（TLS 1.3、WireGuard、Signal）都使用 ECDHE（Elliptic Curve Diffie-Hellman Ephemeral）临时密钥交换，提供前向保密（Forward Secrecy）——即使长期私钥泄露，历史会话也无法解密。

### 核心概念

密钥交换的核心目标：

| 目标 | 说明 |
|------|------|
| **共享密钥协商** | 双方在不安全信道上获得相同密钥 |
| **前向保密** | 长期密钥泄露不影响历史会话 |
| **防止中间人攻击** | 需要认证机制配合 |
| **高效计算** | 椭圆曲线比 RSA 更快 |

### Diffie-Hellman 原理

经典 Diffie-Hellman 的数学原理：

```
DH 密钥交换（简化）：

公共参数：
- 素数 p（大素数）
- 生成元 g

Alice:                  Bob:
1. 选择私钥 a           1. 选择私钥 b
2. 计算公钥 A = g^a mod p  2. 计算公钥 B = g^b mod p
3. 发送 A 给 Bob        3. 发送 B 给 Alice
4. 计算共享密钥          4. 计算共享密钥
   K = B^a mod p           K = A^b mod p
   = g^(ba) mod p          = g^(ab) mod p
   
结果：双方得到相同的 K = g^(ab) mod p
安全性：攻击者知道 g, p, A, B，无法计算 K
       （需要解决离散对数问题）
```

## ECDHE 原理

### 椭圆曲线数学

椭圆曲线密钥交换将 DH 移到椭圆曲线域：

```
椭圆曲线方程：
y² = x³ + ax + b

定义在有限域 GF(p) 上：
- 点 P 是曲线上的点
- 加法运算：P + Q = R（几何意义）
- 标量乘法：kP = P + P + ... + P (k 次)

密钥交换：
- 私钥：随机整数 k
- 公钥：基点 G 的标量乘法 K = kG
- 共享密钥：kA × kB（标量乘法）
```

### ECDHE 流程

ECDHE 临时密钥交换：

```
ECDHE 流程：

客户端                                    服务端
   │                                        │
   │──── 生成临时密钥对                      │──── 生成临时密钥对
   │    私钥 a (随机)                        │    私钥 b (随机)
   │    公钥 A = aG                          │    公钥 B = bG
   │                                        │
   │──────────── 公钥 A ────────────────────>│
   │                                        │
   │<──────────── 公钥 B ────────────────────│
   │                                        │
   │──── 计算共享密钥                        │──── 计算共享密钥
   │    K = aB = abG                         │    K = bA = baG
   │                                        │
   │                                        │
   └── 会话结束，销毁 a, b                  └── 会话结束，销毁 a, b
   
前向保密：a, b 销毁后，无法计算 K
```

### 与静态 DH 的区别

| 特性 | 静态 DH | ECDHE（临时） |
|------|---------|---------------|
| 密钥类型 | 长期密钥 | 每次会话新密钥 |
| 前向保密 | 无 | 有 |
| 计算开销 | 低（可缓存） | 略高（每次计算） |
| 安全性 | 长期密钥泄露风险 | 长期密钥泄露不影响历史 |

## X25519 算法

### Curve25519

X25519 使用 Curve25519 椭圆曲线：

```
Curve25519 参数：
- 級数 p = 2^255 - 19
- 曲线方程：y² = x³ + 486662x² + x
- 基点 G 的 x 坐标 = 9
- 阶数 ~2^252（128 位安全强度）
- 设计者：Daniel J. Bernstein

特点：
- 常量时间实现，抗侧信道攻击
- 无需点验证（设计上保证有效）
- 快速标量乘法
- 公钥格式：32 字节（x 坐标）
```

### X25519 接口

```cpp
// X25519 接口（RFC 7748）

// 生成密钥对
generate_keypair() → (private_key, public_key)
  private_key: 32 字节随机数（经 clamping）
  public_key: 32 字节（X25519(private, basepoint=9)）

// 密钥交换
x25519(private_key, peer_public_key) → shared_key
  shared_key: 32 字节共享密钥

// Clamping（私钥掩码）
private_key[0] &= 248   // 清除低 3 位
private_key[31] &= 127  // 清除最高位
private_key[31] |= 64   // 设置次高位
```

### 安全特性

X25519 的安全特性：

| 特性 | 说明 |
|------|------|
| **128 位安全** | ~2^252 阶数，128 位安全强度 |
| **常量时间** | 所有操作常量时间，抗侧信道 |
| **点验证免费** | 设计上无需显式点验证 |
| **小子群防护** | 无效公钥返回全零共享密钥 |
| **私钥掩码** | Clamping 确保私钥在有效范围 |

### 性能对比

X25519 与其他算法的性能对比：

| 算法 | 计算时间 | 密钥长度 | 安全强度 |
|------|----------|----------|----------|
| X25519 | ~0.1 ms | 32 字节 | 128 位 |
| ECDHE P-256 | ~0.2 ms | 32 字节 | 128 位 |
| RSA-2048 解密 | ~5 ms | 256 字节 | ~112 位 |

X25519 比 RSA 快约 50 倍。

详见 [[ref/crypto/x25519|X25519]]。

## 在 TLS 1.3 中的应用

### TLS 1.3 密钥交换

TLS 1.3 强制使用 ECDHE：

```
TLS 1.3 ClientHello：
┌───────────────────────────────────────────────┐
│ supported_groups: [x25519, secp256r1, ...]    │
│ key_share:                                    │
│   - group: x25519                             │
│   - key_exchange: <32 字节公钥>                │
└───────────────────────────────────────────────┘

TLS 1.3 ServerHello：
┌───────────────────────────────────────────────┐
│ supported_version: 0x0304 (TLS 1.3)          │
│ key_share:                                    │
│   - group: x25519                             │
│   - key_exchange: <32 字节公钥>                │
│ cipher_suite: TLS_AES_128_GCM_SHA256         │
└───────────────────────────────────────────────┘

握手密钥计算：
1. ECDHE 共享密钥 = X25519(客户端私钥, 服务端公钥)
                   = X25519(服务端私钥, 客户端公钥)
2. handshake_secret = HKDF-Extract(0, shared)
3. 派生 client/server handshake key/iv
```

### 前向保密实现

TLS 1.3 前向保密的实现：

```
TLS 1.3 前向保密：

握手阶段：
- 双方生成临时 X25519 密钥对
- 交换公钥（在 ClientHello/ServerHello 中）
- 计算共享密钥
- 会话密钥派生自共享密钥

连接关闭：
- 临时私钥立即销毁
- 历史会话无法解密

长期密钥（证书私钥）：
- 仅用于签名验证
- 不参与密钥交换
- 泄露不影响历史会话
```

详见 [[ref/protocol/tls-handshake|TLS 握手流程]]。

## 在 Prism 中的应用

### Reality 密钥交换

Reality 使用 X25519 进行密钥交换：

```
Reality 密钥交换流程：

1. 服务端配置长期 X25519 密钥对
   - privateKey: 用于计算共享密钥
   - publicKey: 公开给客户端

2. 客户端连接时：
   - 生成临时 X25519 密钥对
   - 在 ClientHello 的 key_share 中发送临时公钥
   - 也包含服务端长期公钥（作为 "shortId"）

3. 服务端响应：
   - 使用长期私钥计算共享密钥
   - 派生会话密钥加密响应

4. 结果：
   - Reality 提供前向保密（客户端临时密钥）
   - 服务端长期密钥泄露不影响历史会话
```

详见 [[core/stealth/reality/handshake|Reality]]。

### SS2022 密钥交换

SS2022 (SIP022) 使用预共享密钥（PSK），不使用 ECDHE：

```
SS2022 密钥模型：

预共享密钥（PSK）：
- 32 字节主密钥
- 客户端和服务端预先配置
- 无前向保密（长期密钥）

会话子密钥派生：
subkey = BLAKE3-derive-key(
    "shadowsocks 2022 session subkey",
    PSK || salt
)

注意：SS2022 不提供前向保密
      长期 PSK 泄露可解密所有历史会话
```

## 安全考量

### 中间人攻击防护

ECDHE 本身不防止中间人攻击，需要认证：

```
中间人攻击：

攻击者位置：
客户端 ── 中间人 ── 服务端

攻击过程：
1. 中间人拦截客户端公钥 A
2. 中间人生成自己的密钥对 (M, M')
3. 中间人发送 M' 给服务端
4. 服务端响应公钥 B
5. 中间人拦截 B，发送 M 给客户端
6. 客户端计算 K1 = aM
7. 中间人计算 K1 = mA（与客户端相同）
8. 服务端计算 K2 = bM'
9. 中间人计算 K2 = M'b（与服务端相同）
10. 中间人可解密和重加密所有消息

防护：使用证书认证服务端公钥
- TLS 1.3: Certificate + CertificateVerify
- 服务端签名证明其公钥真实性
```

### 小子群攻击

小子群攻击的防护：

```
小子群攻击：

攻击场景：
- 椭圆曲线存在小子群（如阶数 n 分解为小因子）
- 攻击者发送小子群中的点作为公钥
- 共享密钥在小子群中，容易猜测

X25519 防护：
- Curve25519 阶数 = 8 × 大素数
- 只有 8 阶小子群（很小）
- 无效公钥（不在正确子群）返回全零共享密钥
- 检测全零即可识别攻击
```

### 实现安全

安全的 X25519 实现：

```cpp
// 安全实现要点：

1. 常量时间
   - 所有计算必须常量时间
   - 不泄露私钥任何信息

2. 私钥生成
   - 使用加密安全随机数生成器
   - 自动应用 clamping

3. 共享密钥验证
   - 检查是否全零（无效公钥）
   - 全零时拒绝连接

4. 私钥销毁
   - 会话结束立即销毁临时私钥
   - 内存清零，不留痕迹
```
